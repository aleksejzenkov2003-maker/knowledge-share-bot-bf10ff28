## 1. Что такое Проект?

**Проект** — это отдельное рабочее пространство для конкретной задачи или клиента. Например:

- Проект для клиента «АВАНТЕРМ» — регистрация товарного знака
- Проект для клиента «РусТрейд» — юридическая экспертиза

Каждый проект содержит:
- **Участников** (люди и AI-агенты)
- **Документы** (файлы, загруженные в проект)
- **Воркфлоу** (автоматизированный процесс обработки задачи)
- **Историю** всех шагов и результатов

### Структура папок

Проекты можно организовывать в **папки** — например, по клиентам или типам услуг:

```
📁 Товарные знаки
  └── 📋 Проект: АВАНТЕРМ
  └── 📋 Проект: Риском

📁 Патенты
  └── 📋 Проект: ООО ТехноПром
```

## 2. Создание нового проекта

### Шаг 1. Открыть раздел «Проекты»

В левом боковом меню нажмите на пункт **«Проекты»**.

Вы увидите список всех ваших проектов (или пустой экран, если проектов ещё нет).

### Шаг 2. Создать проект

Нажмите кнопку **«Создать проект»** (обычно находится в правом верхнем углу или по центру пустого экрана).

### Шаг 3. Заполнить данные

| Поле | Что вводить | Пример |
|------|-------------|--------|
| **Название** | Название клиента или задачи | «АВАНТЕРМ — ТЗ» |
| **Описание** | Краткое описание (необязательно) | «Регистрация словесного знака» |
| **Папка** | Выберите папку из списка | «Товарные знаки» |

### Шаг 4. Сохранить

Нажмите **«Создать»** или **«Сохранить»**. Проект создан!

### Статусы проекта

| Статус | Значение |
|--------|----------|
| 🟢 **Активный** | Проект в работе |
| ✅ **Завершён** | Работа по проекту закончена |
| 📦 **В архиве** | Проект закрыт, но сохранён |

## 3. Добавление участников в проект

Внутри проекта нажмите кнопку **«Добавить»** (иконка человека со знаком «+»).

### Два типа участников

#### 👤 Пользователи (люди)
Коллеги, которые будут видеть проект и работать с результатами.

Роли пользователей:
| Роль | Что может делать |
|------|-----------------|
| **Владелец** | Всё: создать, удалить, пригласить других |
| **Администратор** | Управлять проектом и участниками |
| **Участник** | Работать с воркфлоу, смотреть результаты |
| **Наблюдатель** | Только читать результаты |

#### 🤖 AI-агенты (роли)
Виртуальные ассистенты с заданными инструкциями. Они выполняют этапы воркфлоу автоматически.

Примеры агентов:
- **Досье-аналитик** — собирает информацию о компании
- **Аналитик МКТУ** — анализирует классы товаров
- **QC-Редактор КП** — проверяет качество и структуру документа

## 4. Что такое Воркфлоу (Workflow)?

**Воркфлоу** — это автоматизированный сценарий обработки задачи, состоящий из нескольких этапов (шагов), которые выполняются последовательно (или параллельно).

### Аналогия

Представьте воркфлоу как **конвейер на заводе**:

```
Входные данные → Агент 1 → Агент 2 → Проверка → Итоговый документ
(заявка клиента)   (досье)  (анализ)   (QC)       (КП для клиента)
```

### Воркфлоу «КП по товарному знаку»

Это основной воркфлоу в системе. Он состоит из следующих **этапов**:

```
📥 Входные данные
   ↓
📋 Досье клиента        ← Агент собирает информацию о заявителе
   ↓
✅ Проверка досье       ← Автоматическая проверка полноты данных
   ↓
┌──────────────────────────────────────┐
│  Параллельный анализ (5 агентов)     │
│  • Анализ МКТУ                       │
│  • Охраноспособность знака           │
│  • Конфликтный поиск                 │
│  • Шпион / открытые источники        │
│  • [Дополнительный анализ]           │
└──────────────────────────────────────┘
   ↓
✅ Проверка анализов    ← Проверка полноты всех результатов
   ↓
📄 Итоговое КП         ← Финальный документ для клиента
```

## 5. Запуск воркфлоу в проекте

### Шаг 1. Открыть проект

На странице «Проекты» кликните по нужному проекту.

### Шаг 2. Запустить воркфлоу

Если в проекте ещё нет воркфлоу, вы увидите экран выбора шаблона.

1. Выберите шаблон из списка (например, **«КП по товарному знаку»**)
2. Нажмите **«Создать воркфлоу»**

### Шаг 3. Ввести входные данные

Откроется первый этап — **«Входные данные»**. Введите информацию о клиенте:

```
Контактное лицо: Марат
Телефон / email: +79061159328
Основные товары и услуги: продажа и установка печей и дымоходов

АВАНТЕРМ

Описание знака:
Заявляемое обозначение – словесный товарный знак, выполнен
стандартным шрифтом черного цвета.
```

> **Совет:** Чем подробнее вы укажете данные, тем точнее будут результаты агентов. Укажите:
> - Название компании или бренда
> - Товары и услуги (с классами МКТУ, если известны)
> - ИНН (если есть — поиск по Reputation API будет точнее)
> - Описание товарного знака

### Шаг 4. Нажать «Запустить»

Нажмите кнопку **«Запустить»** на первом этапе. Система начнёт обработку.

## 6. Этапы воркфлоу — как они работают

### Что видно на экране

В верхней части страницы — **полоска прогресса** (например: «7 / 9 этапов»).

Под ней — **полоска этапов** с группами:

```
✅ Входные данные  →  ✅ Досье клиента (2)  →  ⏳ Параллельный анализ (5)  →  ⏸ Финализация
```

Кликая по группе, вы видите шаги внутри неё.

### Статусы этапа

| Иконка | Статус | Значение |
|--------|--------|----------|
| ⏳ | **Ожидает** | Ещё не запущен |
| ▶️ | **Выполняется** | Агент работает прямо сейчас |
| ✅ | **Завершён** | Результат получен |
| ❌ | **Ошибка** | Что-то пошло не так |
| ⏸ | **Ожидает подтверждения** | Требует вашего нажатия «Подтвердить» |

### Вкладки внутри этапа

Каждый этап имеет три вкладки:

| Вкладка | Что показывает |
|---------|---------------|
| **💬 Чат** | Живой поток сообщений от агента в процессе работы |
| **📄 Результат** | Готовый текст / таблица / документ |
| **{ } JSON** | Техническая структура данных (для разработчиков) |

### Управление этапом

| Кнопка | Действие |
|--------|----------|
| ▶️ **Запустить** | Запустить этот этап |
| 🔄 **Перезапустить** | Запустить заново (результат будет перезаписан) |
| ✅ **Подтвердить и продолжить** | Одобрить результат, перейти к следующему этапу |
| ⏭ **Пропустить** | Пропустить этот этап (если не нужен) |

## 7. Работа с этапом «Досье клиента»

Это первый и ключевой этап. Агент **Досье-аналитик** собирает информацию о клиенте.

### Что делает агент

1. **Анализирует** введённые данные (название бренда, товары, услуги)
2. **Ищет компанию** в базе данных Reputation API по названию (или ИНН, если указан)
3. **Формирует досье** с ключевыми данными:
   - ИНН, ОГРН (если найдены)
   - Адрес, статус компании
   - Основная деятельность
   - Зарегистрированные товарные знаки
4. **Проверяет** наличие репутационных данных

### Как это выглядит

В чате вы увидите поток текста — агент описывает, что нашёл. Когда он завершит, во вкладке **«Результат»** появится готовое досье в структурированном виде:

```markdown
## Досье: АВАНТЕРМ

### Общие сведения
- Полное наименование: [если найдено в реестре]
- ИНН: [если найден]
- Деятельность: продажа и установка печей и дымоходов

### Контактные данные
- Контактное лицо: Марат
- Телефон: +79061159328

### Товарный знак
- Заявляемое обозначение: АВАНТЕРМ
- Тип: Словесный
- Описание: ...
```

> **Важно:** Если ИНН не указан, система попытается найти компанию по названию бренда. Это работает в большинстве случаев, но ИНН даёт более точный результат.

### После завершения этапа «Досье»

Следующим идёт **«Проверка досье»** — автоматический узел, который убеждается, что досье было создано (текст не пустой). После него автоматически запустится **Параллельный анализ**.

---

## 8. Проверка качества (QC-агент)

На каждом ключевом этапе работает **QC-агент** (агент проверки качества) — модель Claude Sonnet 4.6, которая проверяет результат другого агента.

### Что проверяет QC-агент

- ✅ Соответствие задаче этапа
- ✅ Наличие обязательных разделов
- ✅ Деловой стиль и структуру документа
- ✅ Логику и полноту текста

### Три возможных вердикта

| Вердикт | Что происходит |
|---------|---------------|
| ✅ **PASS** | Результат принят, работа продолжается |
| ✏️ **REWRITE** | QC сам переписал текст в правильную структуру |
| ❌ **FAIL** | QC не может исправить сам — нужны данные от пользователя |

Если вердикт **REWRITE**, в чате появится сообщение:

> 🔍 **Проверка качества**: REWRITE  
> ✅ Материал автоматически переписан под структуру КП.

Финальный результат в этом случае — **исправленный** текст от QC-агента.

## 9. Итоговый этап — КП (Коммерческое Предложение)

### Как открыть КП

1. Дождитесь завершения всех этапов
2. Перейдите на этап **«Итоговое КП»**
3. Нажмите вкладку **«Результат»**

### Структура итогового КП

КП состоит из двух частей:

#### Документ для клиента (КП_КЛИЕНТ)
Красиво оформленное коммерческое предложение, содержащее:

```
1. Досье клиента
   • Данные заявителя, контакты, деятельность

2. Анализ МКТУ
   • Таблица классов и товаров/услуг
   • Рекомендуемые классы для регистрации

3. Охраноспособность
   • Оценка знака на соответствие требованиям
   • Вывод: высокая / средняя / низкая

4. Конфликтный поиск
   • Таблица похожих зарегистрированных знаков
   • Степень сходства и риски

5. Мониторинг / Шпион
   • Данные из открытых источников
   • Активность конкурентов

6. Стоимость и сроки
   • Таблица стоимости работ
   • Ориентировочные сроки
```

#### Внутренний отчёт (КП_СОТРУДНИК)
Детальная версия с техническими заметками — для внутреннего использования.

### Редактирование КП

1. В результате этапа нажмите кнопку **✏️ Редактировать**
2. Откроется текстовый редактор с разметкой Markdown
3. Внесите правки вручную
4. Нажмите **«Сохранить»**

### Экспорт в PDF

1. Нажмите кнопку **«Скачать PDF»** или **«Экспорт»**
2. Документ сформируется в виде брендированного PDF
3. Файл сохранится в артефактах проекта

## 10. Редактор шаблонов воркфлоу (для администратора)

> ⚠️ **Этот раздел — для администраторов системы.** Рядовым пользователям редактировать шаблоны не нужно.

### Как открыть редактор

В левом меню перейдите в **«Воркфлоу»** → нажмите **«Редактировать»** рядом с нужным шаблоном или откройте **«Конструктор воркфлоу»**.

### Что вы видите

Открывается **визуальный редактор** (canvas) с узлами (нодами) и связями между ними.

```
[Входные данные] → [Досье] → [Проверка] → [Анализ МКТУ] → [КП]
```

Каждый прямоугольник — это **нода** (шаг воркфлоу).

### Как перемещаться по канвасу

| Действие | Как |
|----------|-----|
| Переместить вид | Зажать пробел + тащить мышью |
| Масштаб | Колёсико мыши |
| Выделить ноду | Кликнуть на неё |
| Переместить ноду | Потащить мышью |
| Мини-карта | Кнопка в правом нижнем углу |

### Добавление новой ноды

1. Нажмите кнопку **«+ Добавить шаг»** (или `+` на панели инструментов)
2. Выберите тип ноды из списка
3. Нода появится на канвасе — перетащите её в нужное место

### Создание связей между нодами

1. Наведите мышь на **правую точку** (handle) одной ноды
2. Появится кружок — потяните его к **левой точке** другой ноды
3. Связь создана! Вы можете настроить **маппинг** (какие данные передаются)

## 11. Типы нод в редакторе

### 📥 Input — Входные данные

**Что делает:** Принимает первоначальные данные от пользователя (название компании, описание знака и т.д.)

**Настройки:**
- `Поля ввода` — перечень полей формы (название, ИНН, товары и услуги и т.д.)
- `Схема данных` — структура, в которой данные передаются дальше

**Когда использовать:** Всегда первая нода в любом воркфлоу.

---

### 🤖 Agent — AI Агент

**Что делает:** Отправляет запрос к языковой модели (Claude, GPT и др.) и получает результат.

**Настройки:**
| Поле | Описание |
|------|----------|
| **Роль агента** | Выберите агента из списка (например, «Досье-аналитик») |
| **Переопределение промта** | Дополнительные инструкции для этого конкретного шага |
| **QC-агент** | Агент, который проверяет результат этого шага |
| **Этап (группа)** | Название группы этапов (например, «Досье клиента») |
| **Требует подтверждения** | Включите, если нужно ручное одобрение перед продолжением |
| **Автозапуск** | Включите, если шаг должен запускаться без нажатия кнопки |

---

### 📄 Output — Итог (сборка)

**Что делает:** Собирает результаты всех предыдущих шагов и формирует финальный документ.

**Настройки:**
| Поле | Описание |
|------|----------|
| **Роль агента** | Агент-сборщик (например, «Финальный KP-сборщик») |
| **QC-агент** | Проверяет итоговый документ перед выдачей |
| **Схема результата** | JSON-структура итогового документа |

**Особенность:** Для нод типа Output автоматически добавляется инструкция «Собери и структурируй результаты всех предыдущих этапов».

---

### 🔀 Condition — Условие IF/ELSE

**Что делает:** Проверяет выполнение условия и направляет воркфлоу по одной из двух веток.

**Пример:** «Если указан ИНН → идти по пути А (детальный поиск), иначе → путь Б (поиск по названию)»

**Настройки:**
| Поле | Описание |
|------|----------|
| **Правило** | Выберите поле и условие (`равно`, `не пусто`, `содержит` и т.д.) |
| **Оператор объединения** | `И` (все правила) / `ИЛИ` (хотя бы одно) |

**Операторы сравнения:**

| Оператор | Значение |
|----------|----------|
| `not_empty` | Поле не пустое |
| `empty` | Поле пустое |
| `eq` | Равно значению |
| `contains` | Содержит текст |
| `gt / lt` | Больше / меньше числа |

---

### 🛡️ Quality Check — Проверка результата

**Что делает:** Проверяет данные по набору правил. Если проверка не пройдена — шаг помечается как «Не пройдена».

**Настройки:**
| Поле | Описание |
|------|----------|
| **Правила** | Набор условий (аналогично Condition) |
| **Объединение** | `И` (все) / `ИЛИ` (любое) |

**Пример:** Проверка досье — правило `content not_empty` (текст досье должен быть заполнен).

### ⚙️ Script — Скрипт

## 12. Настройка агентов (ролей)

Перейдите в меню **«Роли / Агенты»** (Admin → ChatRoles).

### Что такое Роль агента

Роль — это набор настроек AI-агента:
- Его **имя** и **системный промт** (инструкции)
- **Языковая модель** (Claude Sonnet 4.6, Claude Opus, GPT-4 и т.д.)
- Доступ к **базе знаний** (папки с документами)
- Доступ к **внешним API** (Reputation API, веб-поиск)

### Основные агенты в воркфлоу «КП»

| Агент | Роль | Модель |
|-------|------|--------|
| **Досье-аналитик** | Собирает информацию о клиенте, ищет в Reputation API | Claude Sonnet |
| **Аналитик МКТУ** | Анализирует классы МКТУ, даёт рекомендации | Claude Sonnet |
| **Аналитик охраноспособности** | Оценивает шансы регистрации | Claude Sonnet |
| **Аналитик конфликтности** | Ищет похожие знаки | Claude Sonnet |
| **Шпион-аналитик** | Мониторинг открытых источников | Claude Sonnet / Perplexity |
| **QC-Редактор КП** | Проверяет качество и структуру каждого блока | Claude Sonnet 4.6 |
| **Финальный KP-сборщик** | Компонует итоговый документ | Claude Sonnet |

### Как создать нового агента

1. Откройте **«Роли / Агенты»**
2. Нажмите **«Создать роль»**
3. Заполните:
   - **Название** — например, «Анализ прецедентов»
   - **Системный промт** — инструкции для агента (что делать, как отвечать)
   - **Модель** — выберите из списка
   - **Папки знаний** — если агенту нужны документы из базы
4. Нажмите **«Сохранить»**

### Системный промт — что это и как писать

**Системный промт** — это инструкция, которую агент получает перед каждым запросом. Он определяет поведение агента.

**Пример промта для QC-агента:**

```
Ты — эксперт по проверке качества коммерческих предложений в сфере
регистрации товарных знаков.

Твоя задача: проверить результат этапа воркфлоу на:
1. Соответствие структуре документа
2. Полноту обязательных разделов
3. Деловой стиль и грамотность
4. Логику и связность текста

Верни СТРОГО JSON:
{
  "verdict": "PASS" | "REWRITE" | "FAIL",
  "feedback": "краткое объяснение",
  "corrected_output": "исправленный текст или пусто",
  "structure_notes": ["заметка 1", "заметка 2"]
}
```

## 13. Часто задаваемые вопросы (FAQ)

### ❓ Можно ли редактировать результат этапа вручную?

**Да!** В любом завершённом этапе во вкладке «Результат» есть кнопка ✏️ **«Редактировать»**. Вы можете изменить текст и сохранить его. Следующие этапы будут использовать отредактированный вариант.

---

### ❓ Что делать, если агент дал неправильный ответ?

Три варианта:
1. **Уточните задачу** в чате этапа (поле «Уточнение для агента»), нажмите отправить — агент ответит с учётом уточнения
2. **Перезапустите этап** кнопкой 🔄 — агент сгенерирует новый ответ
3. **Отредактируйте вручную** результат во вкладке «Результат»

---

### ❓ Что значит «QC не пройдена»?

Агент проверки качества нашёл проблемы, которые не смог исправить сам (вердикт FAIL). Обычно это значит:
- Нужны дополнительные данные от вас
- Агент получил слишком мало контекста

**Что делать:** Нажмите «Подтвердить и продолжить» (если результат вас устраивает, несмотря на предупреждение) или перезапустите этап с уточнением.

---

### ❓ ИНН компании не найден — это проблема?

**Нет.** Система работает и без ИНН:
- Поиск в Reputation API произойдёт по **названию бренда**
- Агент сформирует досье на основе доступных данных
- В разделе «ИНН» будет указано «не найден в реестре» — это нормально

---

### ❓ Параллельные этапы — они все запускаются одновременно?

**Да.** Все 5 агентов параллельного анализа (МКТУ, охраноспособность, конфликтность, шпион и др.) работают **одновременно**, что ускоряет обработку.

---

### ❓ Как долго длится один воркфлоу?

В среднем:
| Этап | Время |
|------|-------|
| Входные данные | 3–5 секунд |
| Досье клиента | 20–60 секунд (поиск в Reputation API) |
| Параллельный анализ (все 5) | 60–120 секунд |
| Итоговое КП | 30–60 секунд |
| **Итого** | **2–5 минут** |

---

### ❓ Можно ли запустить только один этап, а не весь воркфлоу?

**Да.** Нажмите **▶️ Запустить** на конкретном этапе. Остальные запустятся по порядку, или вы можете управлять ими вручную.

---

### ❓ Как скачать итоговое КП?

В этапе «Итоговое КП» нажмите кнопку **«Скачать PDF»** или **«Экспорт»**. Документ будет сохранён в артефактах проекта.

## 14. Возможные проблемы и решения

### ⚠️ «❌ По вашему запросу ничего не найдено»

**Причина:** Система не нашла компанию в базе данных Reputation API.

**Решение:**
- Агент Досье-аналитик всё равно должен сформировать досье на основе введённых данных
- Укажите ИНН (если он есть) — это значительно повышает точность поиска
- Перезапустите этап, добавив в уточнение: «ИНН не известен, используй данные из формы»

---

### ⚠️ «Проверка не пройдена: Правило 2 (inn): не выполнено»

**Причина:** Старое правило проверки требовало наличие ИНН.

**Решение:** Это правило исправлено в последнем обновлении системы. Убедитесь, что на бэкенде (Supabase) применена миграция `20260413140000_fix_dossier_quality_check_rules.sql`. После этого проверка досье будет требовать только наличие текста, а не ИНН.

---

### ⚠️ Этап завис со статусом «Выполняется»

**Причина:** Агент не ответил вовремя (timeout) или произошла ошибка на сервере.

**Решение:**
1. Подождите 2–3 минуты (Reputation API может отвечать медленно)
2. Нажмите **🔄 Перезапустить** на зависшем этапе

---

### ⚠️ Таблицы не отображаются в результатах

**Причина:** Markdown-таблицы не всегда поддерживаются стандартным рендерером.

**Решение:** Убедитесь, что в компонентах результатов используется `remark-gfm` (это уже исправлено в текущей версии системы). Если проблема сохраняется — обратитесь к администратору.

---

### ⚠️ Не работает прокрутка в чате этапа

**Решение:** Это исправлено в текущей версии. Если проблема сохраняется — обновите страницу (F5).
