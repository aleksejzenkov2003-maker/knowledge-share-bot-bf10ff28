1. Что такое Проект?

Проект — это отдельное рабочее пространство для конкретной задачи или клиента. Например:

  • Проект для клиента «АВАНТЕРМ» — регистрация товарного знака
  • Проект для клиента «РусТрейд» — юридическая экспертиза

Каждый проект содержит:

  • 👥 Участников — люди и AI-агенты
  • 📁 Документы — файлы, загруженные в проект
  • ⚙️ Воркфлоу — автоматизированный процесс обработки задачи
  • 📜 История — все шаги и результаты

Структура папок

Проекты можно организовывать в папки — например, по клиентам или типам услуг:

📁 Товарные знаки
  └── 📋 Проект: АВАНТЕРМ
  └── 📋 Проект: Риском

📁 Патенты
  └── 📋 Проект: ООО ТехноПром

✅ ЗавершёнРабота по проекту закончена
📦 В архивеПроект закрыт, но сохранён

2

Кнопка находится в правом верхнем углу или по центру пустого экрана.

3

🤖 AI-агенты (роли)

Виртуальные ассистенты с заданными инструкциями. Они выполняют этапы воркфлоу автоматически.

Примеры агентов:

  • Досье-аналитик — собирает информацию о компании
  • Аналитик МКТУ — анализирует классы товаров
  • QC-Редактор КП — проверяет качество и структуру документа

4. Что такое Воркфлоу (Workflow)?

Воркфлоу — это автоматизированный сценарий обработки задачи, состоящий из нескольких этапов, которые выполняются последовательно (или параллельно).

💡
Аналогия Представьте воркфлоу как конвейер на заводе: каждый агент выполняет свою часть работы и передаёт результат следующему.

Воркфлоу «КП по товарному знаку»

📥 Входные данные ↓
📋 Досье клиента ← Агент собирает информацию о заявителе ↓
✅ Проверка досье ← Автоматическая проверка полноты ↓
⚡ Параллельный анализ (5 агентов одновременно)
   • Анализ МКТУ    • Охраноспособность знака
   • Конфликтный поиск    • Шпион / открытые источники
↓
✅ Проверка анализов ← Проверка полноты результатов ↓
📄 Итоговое КП ← Финальный документ для клиента

5. Запуск воркфлоу в проекте

1

Открыть проект

На странице «Проекты» кликните по нужному проекту.

2

Выбрать шаблон

Если воркфлоу ещё нет — выберите «КП по товарному знаку» и нажмите «Создать воркфлоу».

3

Ввести входные данные

Откроется первый этап. Введите информацию о клиенте в свободной форме:

Контактное лицо: Марат
Телефон / email: +79061159328
Основные товары и услуги: продажа и установка печей и дымоходов

АВАНТЕРМ

Описание знака:
Заявляемое обозначение – словесный товарный знак,
выполнен стандартным шрифтом черного цвета.
✅
Советы по вводу данных Чем подробнее — тем точнее результат. Укажите название бренда, товары/услуги, ИНН (если есть), описание знака (словесный/изобразительный/комбинированный).
4

Нажать «Запустить»

Система начнёт обработку. Прогресс виден в верхней полоске этапов.

7. Работа с этапом «Досье клиента»

Это первый и ключевой этап. Агент Досье-аналитик собирает информацию о клиенте.

Что делает агент

  1. Анализирует введённые данные (название бренда, товары, услуги)
  2. Ищет компанию в базе Reputation API по названию (или ИНН, если указан)
  3. Формирует досье с ключевыми данными: ИНН, ОГРН, адрес, статус компании, деятельность, товарные знаки
  4. Передаёт результат на проверку качества (QC-агент)

Пример готового досье

## Досье: АВАНТЕРМ

### Общие сведения
- Полное наименование: [если найдено в реестре]
- ИНН: [если найден]
- Деятельность: продажа и установка печей и дымоходов

### Контактные данные
- Контактное лицо: Марат
- Телефон: +79061159328

### Товарный знак
- Заявляемое обозначение: АВАНТЕРМ
- Тип: Словесный
ℹ️
Если ИНН не указан Система найдёт компанию по названию бренда. Это работает в большинстве случаев, но ИНН даёт более точный результат.

8. Проверка качества (QC-агент)

На каждом ключевом этапе работает QC-агент (модель Claude Sonnet 4.6), которая проверяет результат основного агента.

Что проверяет QC-агент

  • ✅ Соответствие задаче этапа
  • ✅ Наличие обязательных разделов
  • ✅ Деловой стиль и структуру документа
  • ✅ Логику и полноту текста

Три возможных вердикта

ВердиктЧто происходит
✅ PASSРезультат принят, работа продолжается автоматически
✏️ REWRITEQC сам переписал текст в правильную структуру — финальный результат уже исправлен
❌ FAILQC не может исправить сам — нужны данные от пользователя
✅
Что делать при FAIL? Нажмите «Подтвердить и продолжить» (если результат устраивает) или перезапустите этап, добавив уточнение в чате.

Структура итогового КП

КП состоит из двух частей:

Документ для клиента (КП_КЛИЕНТ)

РазделСодержание

10. Редактор шаблонов воркфлоу (для администратора)

⚠️
Только для администраторов Рядовым пользователям редактировать шаблоны не нужно. Этот раздел для тех, кто настраивает систему.

Как открыть редактор

В левом меню перейдите в «Воркфлоу» → нажмите «Редактировать» рядом с нужным шаблоном.

Что вы видите

Открывается визуальный редактор (canvas) с прямоугольниками (нодами) и стрелками между ними.

Навигация по канвасу

ДействиеКак
Переместить видЗажать пробел + тащить мышью
МасштабКолёсико мыши
Выделить нодуКликнуть на неё
Переместить нодуПотащить мышью
Удалить ноду/связьВыделить + Delete

Создание связей между нодами

Наведите мышь на правую точку одной ноды, появится кружок — потяните его к левой точке другой ноды. Связь создана!

Группировка по этапам

В настройках каждой ноды укажите поле «Этап (группа)» — например, «Досье клиента», «Параллельный анализ». Ноды с одинаковой группой визуально объединятся в рамку на канвасе.

11. Типы нод в редакторе

📥
Input — Входные данные
input

Что делает: Принимает первоначальные данные от пользователя.

Настройки: Поля формы ввода (название бренда, ИНН, товары и услуги и т.д.)

Когда использовать: Всегда первая нода в любом воркфлоу.

🤖
Agent — AI Агент
agent

Что делает: Отправляет запрос к языковой модели и получает результат.

НастройкаОписание
Роль агентаВыберите агента (например, «Досье-аналитик»)
Переопределение промтаДополнительные инструкции для этого шага
QC-агентАгент, проверяющий результат этого шага
Этап (группа)Название группы (например, «Досье клиента»)
АвтозапускШаг запускается без нажатия кнопки
📄
Output — Итог (сборка)
output

Что делает: Собирает результаты всех предыдущих шагов и формирует финальный документ.

Автоматически получает инструкцию: «Собери и структурируй результаты всех предыдущих этапов в финальный документ».

🔀
Condition — Условие IF/ELSE
condition

Что делает: Проверяет условие и направляет воркфлоу по одной из двух веток.

Пример: «Если указан ИНН → детальный поиск, иначе → поиск по названию»

ОператорЗначение
not_emptyПоле не пустое
emptyПоле пустое
eqРавно значению
containsСодержит текст
gt / ltБольше / меньше числа
🛡️
Quality Check — Проверка результата
quality_check

Что делает: Проверяет данные по набору правил. Если проверка не пройдена — шаг помечается соответственно.

Пример: Проверка досье — правило content not_empty (текст досье должен быть заполнен).

⚙️
Script — Скрипт
script

Что делает: Вызывает стороннюю функцию (например, сканер открытых источников, парсер сайтов).

Пример функции: spy-market-scan — мониторинг рынка и конкурентов.

13. Часто задаваемые вопросы (FAQ)

❓ Можно ли редактировать результат этапа вручную?
Да! В любом завершённом этапе во вкладке «Результат» есть кнопка ✏️ «Редактировать». Следующие этапы будут использовать отредактированный вариант.
❓ Что делать, если агент дал неправильный ответ?
Три варианта:
1. Уточните задачу в поле «Уточнение для агента» и отправьте
2. Перезапустите этап кнопкой 🔄
3. Отредактируйте вручную во вкладке «Результат»
❓ Что значит «QC не пройдена»?
QC-агент нашёл проблемы, которые не смог исправить сам (вердикт FAIL). Нажмите «Подтвердить и продолжить» (если результат устраивает) или перезапустите этап с уточнением.
❓ ИНН компании не найден — это проблема?
Нет. Система работает и без ИНН — поиск произойдёт по названию бренда. В разделе «ИНН» будет указано «не найден в реестре» — это нормально.
❓ Как долго длится один воркфлоу?
В среднем 2–5 минут:
• Входные данные: 3–5 сек
• Досье клиента: 20–60 сек (поиск в Reputation API)
• Параллельный анализ (все 5): 60–120 сек
• Итоговое КП: 30–60 сек
❓ Можно ли запустить только один этап?
Да. Нажмите ▶️ «Запустить» на конкретном этапе. Остальные можно запустить потом или вручную по порядку.
❓ Параллельные этапы — они все запускаются одновременно?
Да! Все 5 агентов параллельного анализа работают одновременно, что значительно ускоряет обработку.
❓ Как скачать итоговое КП?
В этапе «Итоговое КП» нажмите кнопку «Скачать PDF» или «Экспорт». Документ сохранится в артефактах проекта.

14. Возможные проблемы и решения

❌
«По вашему запросу ничего не найдено» Система не нашла компанию в Reputation API. Агент Досье-аналитик всё равно сформирует досье на основе введённых данных. Для точности укажите ИНН.
⚠️
«Проверка не пройдена: Правило (inn): не выполнено» Старое правило требовало ИНН. Исправлено в обновлении — убедитесь, что в Supabase применена миграция 20260413140000_fix_dossier_quality_check_rules.sql.
⚠️
Этап завис со статусом «Выполняется» Подождите 2–3 минуты (Reputation API может отвечать медленно), затем нажмите 🔄 «Перезапустить».
ℹ️
Таблицы не отображаются в результатах Убедитесь, что в браузере не устаревший кеш. Обновите страницу (Ctrl+Shift+R).
