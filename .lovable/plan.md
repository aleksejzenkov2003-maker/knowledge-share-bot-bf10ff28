

# Анализ аудио — план реализации

## Что будет сделано

Новый раздел «Анализ аудио» доступный всем пользователям. Пользователь загружает аудиофайл → Whisper (OpenAI) транскрибирует → результат отображается → пользователь выбирает агента из существующих `chat_roles` и пишет запрос по транскрипту → ответ агента через `chat-stream` → вся история сохраняется в БД.

## Шаги реализации

### 1. Создать таблицы в БД (миграция)

- **`audio_sessions`** — сессии анализа аудио:
  - `id`, `user_id`, `title`, `audio_file_path`, `audio_file_name`, `transcript`, `status` (uploading/transcribing/ready/error), `created_at`, `updated_at`
  
- **`audio_session_messages`** — сообщения чата по транскрипту:
  - `id`, `session_id` (FK → audio_sessions), `user_id`, `role_id` (FK → chat_roles, nullable), `message_role` (user/assistant), `content`, `metadata` (jsonb), `created_at`

- RLS: пользователи видят только свои сессии и сообщения.

### 2. Edge Function `audio-transcribe`

- Принимает аудиофайл (multipart/form-data)
- Отправляет в OpenAI Whisper API (`https://api.openai.com/v1/audio/transcriptions`) используя `OPENAI_API_KEY` (уже есть в секретах)
- Поддержка форматов: mp3, mp4, mpeg, mpga, m4a, wav, webm, ogg
- Возвращает текст транскрипции
- Сохраняет транскрипт в `audio_sessions`

### 3. Страница `src/pages/AudioAnalysis.tsx`

**Левая панель (сайдбар):**
- Список сессий пользователя (как в чатах)
- Кнопка «Новый анализ»
- Удаление / переименование сессий

**Основная область — три фазы:**

1. **Загрузка аудио** — drag & drop зона + кнопка выбора файла, индикатор прогресса, статус транскрибации
2. **Транскрипт** — отображение текста с возможностью копирования
3. **Чат по транскрипту** — выбор агента из `chat_roles`, поле ввода запроса, отправка через существующий `chat-stream` (транскрипт добавляется в контекст как системное сообщение), стриминг ответов

### 4. Роутинг и навигация

- Маршрут `/audio-analysis` в App.tsx, доступен всем авторизованным (без ограничения ролей)
- Пункт «Анализ аудио» в сайдбаре (`AdminSidebar.tsx`) в секции «Основное» с иконкой `Mic`

### 5. Хук `useAudioAnalysis.ts`

- Управление сессиями (CRUD)
- Загрузка аудио в Storage → вызов `audio-transcribe`
- Отправка сообщений агенту через `chat-stream` с транскриптом в контексте
- Стриминг ответов (SSE)

## Технические детали

- **Whisper API**: используем `OPENAI_API_KEY` (уже в секретах), модель `whisper-1`, лимит файла 25MB
- **Storage bucket**: `audio-files` для хранения загруженных аудио
- **Контекст агента**: при отправке запроса агенту, транскрипт передается как часть `message_history` с пометкой «Транскрипт аудиозаписи»
- **Стриминг**: переиспользуем существующую логику SSE из `chat-stream`
