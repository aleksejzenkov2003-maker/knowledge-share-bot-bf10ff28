

# Улучшить поиск компаний в Reputation

## Проблема
При поиске по короткому названию ("Скат") API возвращает ~20 результатов, и нужная компания может не попасть в выдачу. API Reputation.ru не поддерживает фильтрацию по адресу на стороне сервера.

## Решение

### 1. Подсказка для пользователя
Добавить информационный блок под полем поиска с советами:
- "Для точного поиска используйте ИНН или ОГРН"
- "Можно добавить город к названию: Скат Санкт-Петербург"

### 2. Поле поиска с примерами (placeholder)
Изменить placeholder на: `ИНН, ОГРН или название (+ город для точности)`

### 3. Фильтр по адресу в результатах
Добавить текстовый фильтр по адресу на фронтенде — пользователь сможет ввести "Чугунная" или "Санкт-Петербург" и отфильтровать результаты локально.

## Технические детали

### Файл: `src/pages/Reputation.tsx`

**1. Добавить текстовый фильтр по адресу:**
- Новый state `filterAddress` (string)
- Input-поле в блоке фильтров рядом с существующими фильтрами по статусу и городу
- В `filteredResults` добавить проверку: если `filterAddress` не пуст, фильтровать результаты по вхождению текста в `r.Address` (case-insensitive)

**2. Обновить placeholder поля поиска:**
- Заменить текущий placeholder на `ИНН, ОГРН или название компании (+ город)`

**3. Добавить подсказку:**
- Небольшой текст под полем поиска: "Совет: для точного поиска используйте ИНН или ОГРН. К названию можно добавить город."
- Показывать только когда поле поиска пустое или при первом визите
