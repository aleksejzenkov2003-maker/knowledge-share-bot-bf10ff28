

# Диагностика: Агент «Иследование (1)» не работает

## Найденная проблема

Агент «Иследование (1)» настроен на модель `sonar-deep-research` (Perplexity). Эта модель выполняет многошаговое исследование и может работать до **5 минут** (300 секунд).

**Корневая причина:** Edge-функция на хостинге имеет ограничение по времени выполнения (~150 секунд). Функция отправляет запрос к Perplexity API, но не успевает дождаться ответа — платформа принудительно завершает (shutdown) функцию раньше, чем `sonar-deep-research` отвечает.

Подтверждение из логов:
- 07:25:09 — запрос отправлен (`Perplexity request: model=sonar-deep-research`)
- Нет логов с ответом (`Perplexity response status` отсутствует)
- 07:26:24 — принудительный shutdown функции

Дополнительно: в таблице `ai_providers` у Perplexity **пустой api_key** — ключ берётся из переменной окружения `PERPLEXITY_API_KEY` (через коннектор), что работает, но неочевидно.

## План исправления

### 1. Вынести `sonar-deep-research` в отдельную edge-функцию с увеличенным таймаутом

Создать `supabase/functions/deep-research/index.ts`:
- Единственная задача — вызвать `sonar-deep-research` без стриминга
- Вернуть результат как обычный JSON-ответ
- Добавить в `config.toml` настройку `verify_jwt = false`

### 2. Изменить фронтенд: для `sonar-deep-research` использовать non-streaming fetch с увеличенным клиентским таймаутом

В `useOptimizedChat.ts` (и аналогичных хуках):
- Определять, что выбранная роль использует `sonar-deep-research`
- Для неё вызывать `deep-research` функцию вместо `chat-stream`
- Установить клиентский таймаут 360 секунд (AbortController)
- Показывать пользователю индикатор "Исследование может занять до 5 минут..."

### 3. Альтернативный (простой) вариант

Если проблема в wall-clock лимите платформы, который нельзя увеличить:
- Переключить агент «Иследование (1)» на модель `sonar-pro` (ответ за 5-15 секунд, тоже с веб-поиском)
- `sonar-deep-research` оставить только для случаев, когда доступен более длинный таймаут

### Технические детали

**Файлы для изменения:**
- `supabase/functions/chat-stream/index.ts` — добавить логирование таймаута для Perplexity
- `src/hooks/useOptimizedChat.ts` — увеличить клиентский таймаут для deep-research ролей
- `src/hooks/useChat.ts` — аналогично
- SQL: заполнить `api_key` в `ai_providers` для Perplexity (чтобы не зависеть от env fallback)

**Вопрос для решения:** Предпочитаете простой вариант (переключить на `sonar-pro`) или создать отдельную функцию для deep-research?
