

# Исправление классификации объектов интеллектуальной собственности

## Проблема

API Reputation.ru возвращает все объекты интеллектуальной собственности (товарные знаки, патенты, промышленные образцы, программы для ЭВМ) через один эндпоинт. Каждый объект имеет поле `Registry`, определяющее его тип:

| Registry | Тип объекта | ФИПС DB |
|----------|------------|---------|
| `RUTM` | Товарный знак | RUTM |
| `RUTMAP` | Заявка на ТЗ | RUTMAP |
| `RUPM` | Полезная модель | RUPM |
| `RUPMAP` | Заявка на полезную модель | RUPMAP |
| `RUDE` | Промышленный образец | RUDE |
| `RUDEAP` | Заявка на пром. образец | RUDEAP |
| `EVM` | Программа для ЭВМ | RSPODB |

Сейчас система:
1. Все объекты называет "Товарные знаки" (ТЗ)
2. Для всех генерирует ссылку на ФИПС с `DB=RUTM`, из-за чего номер патента/пром. образца ведет на чужой товарный знак с тем же номером
3. Не различает типы объектов визуально

## Решение

### 1. Переименовать раздел с "Товарные знаки" на "Интеллектуальная собственность"

В массиве `DATA_SECTIONS` и во всех местах, где упоминается "Товарные знаки" как заголовок раздела.

### 2. Добавить маппинг Registry -> тип и ссылка ФИПС

Создать в `Reputation.tsx` вспомогательную функцию, которая по полю `Registry` определяет:
- Человекочитаемое название типа (Товарный знак, Промышленный образец, Полезная модель, Программа для ЭВМ, Заявка на ТЗ и т.д.)
- Правильный параметр `DB` для ссылки на ФИПС
- Иконку/бейдж для визуального различия

```text
Registry -> Label / DB
RUTM     -> "Товарный знак" / DB=RUTM
RUTMAP   -> "Заявка на ТЗ" / DB=RUTMAP  
RUPM     -> "Полезная модель" / DB=RUPM
RUPMAP   -> "Заявка на ПМ" / DB=RUPMAP
RUDE     -> "Пром. образец" / DB=RUDE
RUDEAP   -> "Заявка на ПО" / DB=RUDEAP
EVM      -> "Программа ЭВМ" / DB=RSPODB
```

### 3. Исправить генерацию ссылки на ФИПС

Заменить захардкоженную строку:
```
DB=RUTM
```
на динамическую, использующую `Registry` объекта.

Для заявок (`_source === 'applications'`) использовать `PatentRegistry` или `ApplicationRegistry` если `Registry` указывает на заявку.

### 4. Добавить бейдж типа на каждую карточку

Вместо единого значка "#ТЗ" показывать тип объекта (Товарный знак, Патент, Пром. образец, Программа ЭВМ) с соответствующим бейджем.

### Технические детали

Все изменения в одном файле: `src/pages/Reputation.tsx`

- Строки 41-43: переименовать секцию
- Строки 1030-1085: обновить рендеринг карточек -- использовать `Registry` для определения типа, корректной ссылки ФИПС и отображения бейджа типа
- Добавить вспомогательную функцию `getRegistryInfo(registry: string)` для маппинга
