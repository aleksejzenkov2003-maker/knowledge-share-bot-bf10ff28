

## Plan: Парсинг данных с ФИПС по номеру регистрации

### Что делаем
Добавляем кнопку "Загрузить с ФИПС" в таблице товарных знаков. При нажатии — edge function парсит HTML-страницу ФИПС, извлекает все данные (правообладатель, даты, классы МКТУ, изображение знака и т.д.) и обновляет запись в БД.

### Структура HTML ФИПС (проверено)
Страница имеет предсказуемую структуру:
- Поля размечены кодами ВОИС в скобках: `(111)` — номер регистрации, `(732)` — правообладатель, `(511)` — классы МКТУ, `(540)` — изображение
- Изображение ТЗ — тег `<img>` внутри блока `(540)`, ссылка на JPG на `fips.ru/Archive/...`
- Статус — в элементе с классом `.Status`

### Реализация

**1. Edge function `supabase/functions/fips-parse/index.ts`**
- Принимает `registration_number`
- Делает `fetch` на `https://fips.ru/registers-doc-view/fips_servlet?DB=RUTM&DocNumber={number}&TypeFile=html`
- Парсит HTML регулярками (Deno не имеет DOM-парсера из коробки, но структура простая):
  - `(732)` → правообладатель
  - `(151)` → дата регистрации
  - `(511)` → классы МКТУ
  - `(540)` img src → URL изображения
  - Статус (действует/не действует)
  - Дата истечения `(181)`
  - И другие доступные поля
- Возвращает JSON с извлечёнными данными + URL изображения

**2. Обновление `src/pages/Trademarks.tsx`**
- Добавить кнопку "Загрузить с ФИПС" (иконка Download) в строке таблицы рядом с кнопкой "Открыть в ФИПС"
- При нажатии — вызов edge function, показ результата в диалоге с превью (включая изображение ТЗ)
- Кнопка "Сохранить" в диалоге обновляет запись в `trademarks`
- Также добавить массовую кнопку "Обогатить выбранные" (опционально, на будущее)

**3. Извлекаемые поля**

| Код ВОИС | Поле | Маппинг в БД |
|----------|------|-------------|
| (111) | Номер регистрации | `registration_number` |
| (151) | Дата регистрации | `registration_date` |
| (181) | Срок действия | metadata |
| (732) | Правообладатель | `right_holder_name` |
| (511) | Классы МКТУ | `description_element` / metadata |
| (540) | Изображение | сохраняем URL в metadata |
| Status | Статус | `actual` |

### Ограничения
- ФИПС может блокировать частые запросы — добавим задержку и обработку ошибок
- Изображение хранится как внешний URL (не скачиваем в storage, чтобы не нарушать авторские права)
