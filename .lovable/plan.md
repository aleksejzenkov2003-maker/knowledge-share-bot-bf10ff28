

# Визуальный редактор Workflow (n8n-style) + уход от чата

## Суть изменения

Убираем режим «Чат» из проектов. Проект = Workflow. Добавляем визуальный редактор шаблонов workflow в стиле n8n — drag-and-drop блоки (ноды), связи между ними, настройка промптов и агентов прямо на канвасе.

## Что будет сделано

### 1. Установка @xyflow/react (React Flow)

Библиотека для визуальных node-based графов. Открытая, MIT-лицензия, стандарт для таких задач.

### 2. Страница «Редактор Workflow» (`/workflow-templates`)

Новая admin-страница для создания и редактирования шаблонов workflow.

**Канвас (React Flow):**
- Каждый шаг шаблона — нода на канвасе
- Ноды соединяются линиями (edges) по порядку шагов
- Drag-and-drop для перемещения нод
- Кнопка «+ Добавить шаг» — создаёт новую ноду

**Панель настроек ноды (правая боковая панель при клике на ноду):**
- Название шага
- Описание
- Выбор агента (из `chat_roles`)
- Системный промпт агента (текстовое поле)
- `is_user_editable` — переключатель
- `auto_run` — переключатель
- Input/Output schema (JSON-редактор)

**Типы нод:**
- «Ввод данных» (начальная нода — пользователь заполняет форму)
- «AI Агент» (основная нода — агент выполняет задачу)
- «Итог» (финальная нода — сборка результата)

### 3. Компоненты визуального редактора

- `WorkflowCanvas` — обёртка React Flow с нодами и связями
- `WorkflowNode` — кастомная нода (отображает название, иконку агента, статус)
- `WorkflowNodeConfigPanel` — правая панель для настройки выбранной ноды
- `WorkflowTemplateList` — список шаблонов с кнопкой «Создать»
- `AddNodeMenu` — меню добавления новой ноды

### 4. Изменения в ProjectChat → ProjectWorkflow

- Убираем toggle «Чат / Workflow» — остаётся только Workflow
- Убираем чат-интерфейс, всё через `WorkflowPanel`
- Убираем sidebar с чатами — заменяем на sidebar с этапами workflow
- Упрощаем хедер проекта

### 5. Расширение таблицы `workflow_template_steps`

Миграция для добавления:
- `prompt_override` TEXT — кастомный промпт для шага (поверх системного промпта агента)
- `position_x` FLOAT, `position_y` FLOAT — координаты ноды на канвасе
- `node_type` TEXT DEFAULT 'agent' — тип ноды ('input', 'agent', 'output')

### 6. Навигация

- Новый пункт в admin sidebar: «Workflow шаблоны» (`/workflow-templates`) — только для admin
- Маршрут в `App.tsx`: `/workflow-templates` и `/workflow-templates/:id` (редактор конкретного шаблона)

### 7. Хук `useWorkflowEditor`

Управление состоянием редактора:
- Загрузка/сохранение шаблона и шагов
- CRUD шагов (добавить/удалить/обновить ноду)
- Пересчёт `step_order` при изменении связей
- Синхронизация позиций нод с БД
- Валидация графа (все ноды связаны, есть входная и выходная)

## Порядок реализации

1. **Миграция БД** — новые поля в `workflow_template_steps`
2. **Установка `@xyflow/react`**
3. **Компоненты редактора** — `WorkflowCanvas`, `WorkflowNode`, `WorkflowNodeConfigPanel`
4. **Страница шаблонов** — список + редактор
5. **Хук `useWorkflowEditor`**
6. **Рефакторинг ProjectChat** — убрать чат, оставить только workflow
7. **Навигация** — sidebar + роуты

## Технические детали

- React Flow хранит позиции нод клиентски, при сохранении записываются в `position_x/y` в БД
- Связи (edges) вычисляются из `step_order` — не хранятся отдельно
- Промпт шага (`prompt_override`) отправляется в `workflow-step-execute` как дополнение к системному промпту агента
- Список доступных агентов подгружается из `chat_roles`
