

# Убрать лимит 10 результатов в Reputation API

## Проблема
В edge-функции `reputation-api` результаты поиска обрезаются до 10 (`slice(0, 10)`) на строке 186, а при единственном совпадении до 5 (`slice(0, 5)`) на строке 209. Из-за этого пользователь видит максимум 10 компаний даже если API Reputation.ru вернул больше.

## Решение

### 1. Edge-функция `supabase/functions/reputation-api/index.ts`
- Строка 186: убрать `slice(0, 10)` — отдавать все результаты
- Строка 209: убрать `slice(0, 5)` — отдавать все результаты

### 2. Фронтенд `src/pages/Reputation.tsx`
- Добавить пагинацию или кнопку "Показать ещё" для случаев, когда результатов много (20+)
- Показывать счётчик "Найдено N компаний"

## Технические детали

**Файл: `supabase/functions/reputation-api/index.ts`**
- Строка 186: `results.slice(0, 10)` заменить на `results`
- Строка 209: `results.slice(0, 5)` заменить на `results`

**Файл: `src/pages/Reputation.tsx`**
- Добавить state `visibleCount` (начальное значение 20)
- Показывать `filteredResults.slice(0, visibleCount)`
- Кнопка "Показать ещё" увеличивает `visibleCount` на 20
- Счётчик "Найдено X компаний" в заголовке блока результатов
