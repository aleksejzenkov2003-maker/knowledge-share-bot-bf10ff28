

## План: Исправить regex для ФИО — \b не работает с кириллицей

### Проблема

Регулярное выражение для полного ФИО использует `\b` (word boundary) в конце паттерна:

```
/[А-ЯЁ][а-яё]{1,20}[\s\u00A0]+[А-ЯЁ][а-яё]{1,15}[\s\u00A0]+[А-ЯЁ][а-яё]*(?:ович[ауем]?|...)\b/g
```

В JavaScript/Deno `\b` определяется через `\w`, который включает **только** `[A-Za-z0-9_]`. Кириллические буквы НЕ являются "word characters". Поэтому `\b` после кириллической буквы (например "ч" в "Иванович") **всегда ложно** -- потому что и "ч", и следующий пробел оба считаются "не-word", и граница слова между ними отсутствует.

Результат: **ни одно ФИО не маскируется**.

Подтверждено тестом: "Иванов Иван Иванович" -- 0 совпадений.

### Решение

Заменить `\b` на `(?=[^а-яёА-ЯЁ]|$)` -- lookahead, который проверяет что после отчества идёт не-кириллический символ (пробел, запятая, точка, конец строки и т.д.).

### Файл для изменения

**`supabase/functions/_shared/pii-patterns.ts`**, строка 46:

Было:
```
/[А-ЯЁ][а-яё]{1,20}[\s\u00A0]+[А-ЯЁ][а-яё]{1,15}[\s\u00A0]+[А-ЯЁ][а-яё]*(?:ович[ауем]?|евич[ауем]?|овн[аыеой]?|евн[аыеой]?|ичн[аыеой]?)\b/g
```

Станет:
```
/[А-ЯЁ][а-яё]{1,20}[\s\u00A0]+[А-ЯЁ][а-яё]{1,15}[\s\u00A0]+[А-ЯЁ][а-яё]*(?:ович[ауем]?|евич[ауем]?|овн[аыеой]?|евн[аыеой]?|ичн[аыеой]?)(?=[^а-яёА-ЯЁ]|$)/g
```

Единственное изменение: `\b` заменён на `(?=[^а-яёА-ЯЁ]|$)`.
