

# Разбиение больших Excel-файлов на части (как PDF)

## Идея

Использовать тот же механизм, что и для PDF: на клиенте определять, что Excel-файл большой, и разбивать его на "виртуальные части" перед загрузкой. Каждая часть загружается как отдельный документ с `parent_document_id`, `part_number`, `total_parts`.

## Как это работает для PDF сейчас

1. Клиент проверяет размер файла (порог ~4MB)
2. Разбивает PDF на части по 50 страниц через `pdf-lib`
3. Загружает каждую часть как отдельный Blob с метаданными (parent, part_number, total_parts)
4. Сервер обрабатывает каждую часть независимо

## Что нужно сделать для Excel

### 1. Новый файл: `src/components/documents/excelSplitter.ts`

Создать аналог `pdfSplitter.ts` для Excel:
- Функция `splitExcel(file, rowsPerPart)` -- читает XLSX через JSZip, парсит XML листов, считает строки
- Разбивает строки на группы (например, по 5000 строк на часть)
- Генерирует новые XLSX-файлы для каждой части (пересоздаёт XML с подмножеством строк)
- Возвращает массив Blob-ов с метаданными

### 2. Изменения в `src/pages/Documents.tsx`

- Добавить проверку при загрузке: если файл `.xlsx`/`.xls` и размер > 1MB, показать предупреждение о разбиении
- Использовать `splitExcel()` для разбиения
- Загружать части так же, как PDF: первая часть -- parent, остальные ссылаются на неё

### 3. Изменения в `supabase/functions/process-document/index.ts`

- **Убрать `EXCEL_MAX_CHARS` обрезку** (строки 2200-2208) -- больше не нужна, потому что каждая часть будет небольшой
- Каждая часть обрабатывается независимо и полностью

## Техническая сложность: пересборка XLSX

XLSX -- это ZIP-архив с XML-файлами. Чтобы разбить его на части:

```text
1. Распаковать ZIP (JSZip)
2. Найти xl/worksheets/sheet*.xml
3. Распарсить XML, найти все <row> элементы
4. Для каждой части:
   a. Взять строки [start..end]
   b. Пересобрать sheet XML с этими строками
   c. Скопировать sharedStrings.xml, styles.xml и прочие файлы
   d. Упаковать обратно в ZIP
5. Вернуть Blob
```

Заголовки (первая строка) копируются в каждую часть, чтобы парсер мог правильно интерпретировать данные.

## Параметры

- **Порог разбиения**: файлы > 1MB (настраиваемо)
- **Строк на часть**: 5,000 строк (при 5 колонках это ~200-300KB текста, хорошо укладывается в лимиты Edge Function)
- **Максимум частей**: 50 (защита от экстремальных файлов с 250K+ строк)

## Ожидаемый результат

- Файл МКТУ (2.67M символов, ~10,000+ строк) будет разбит на 2-3 части
- Каждая часть обработается полностью без обрезки
- В интерфейсе части отобразятся так же, как части PDF (с badge "1/3", "2/3", "3/3")
- Все записи таблицы попадут в RAG
