
## План: Добавить превью маскирования PII перед загрузкой документа

### Проблема
Текущая архитектура маскирует PII **после обработки** документа, что скрывает от пользователя, какие данные будут скрыты. Пользователь не может увидеть результат маскирования перед тем, как документ попадёт в RAG и AI обработку.

### Решение: Двухступенчатая система

**Шаг 1: Превью маскирования в UI**
- Когда пользователь выбирает файл и включает `contains_pii`, показать превью первых 500-1000 символов текста документа
- Отобразить, какие данные будут замаскированы (с цветной подсветкой)
- Позволить пользователю видеть токены вроде `[PHONE_1]`, `[PERSON_1]`

**Шаг 2: Без изменений в backend логике**
- Маскирование в `process-document` остаётся на месте (происходит после OCR, перед чанкированием)
- Это правильное место, потому что:
  - После извлечения текста (OCR)
  - Перед разбиением на chunks
  - Перед отправкой в embeddings / LLM

### Технические детали

**Frontend (src/pages/Documents.tsx)**
1. Добавить кнопку "Показать превью маскирования" при `contains_pii=true`
2. Открыть modal с:
   - Превью текста документа (первые N символов)
   - Цветная подсветка найденных PII типов
   - Счётчик найденных типов (телефоны, имена, адреса и т.д.)
   - Список примеров замаскированных данных
3. Использовать уже существующую функцию `maskPii` из backend (edge function `pii-mask`)

**Backend (уже существует)**
- Функция `supabase/functions/pii-mask/index.ts` может быть переиспользована для превью
- Можно вызвать её из клиента с `contains_pii=true` перед загрузкой
- Создать специальный mode для "preview only" (без сохранения в БД)

### Архитектура потока
```
1. Пользователь выбирает файл
2. Если `contains_pii=true`:
   - Читаем первые N страниц файла (для preview)
   - Вызываем `pii-mask` с mode="preview"
   - Показываем превью с подсветкой
3. При подтверждении загрузки:
   - Загружаем весь файл
   - Флаг `contains_pii` сохраняется
   - При обработке `process-document` маскирует весь текст

### Файлы для изменения
1. **src/pages/Documents.tsx** - добавить UI для превью
2. **supabase/functions/pii-mask/index.ts** - опционально добавить режим preview
3. **Новый компонент** - `src/components/chat/PiiPreviewDialog.tsx` - реюзабельный диалог превью

### Преимущества
- Пользователь видит ДО загрузки, какие данные будут замаскированы
- Может отменить загрузку или переименовать документ если много ложных срабатываний
- Прозрачность и контроль над процессом 152-ФЗ соответствия
- Улучшает UX для документов с чувствительными данными
