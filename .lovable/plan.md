

# План: Улучшенное форматирование текста источников

## Проблема

Текст чанков в базе данных хранится **без переносов строк** - при извлечении из PDF переносы были потеряны. Функция `formatChunkContent` ищет `\n`, но их нет в данных.

**Пример из БД:**
```
...утвержденным постановлением Правительства Российской Федерации от 10 декабря 2008 г. № 941 (Собрание законодательства Российской Федерации, 2008, № 51, ст.6170; 2020, N 23, ст.3861) (далее Положение о пошлинах). (Пункт в редакции...
```

Всё идёт сплошным текстом без `\n`.

## Решение

Добавить **эвристический парсинг** для вставки переносов строк на основе:
1. Точка + пробел + заглавная буква → новый абзац (с исключениями)
2. Нумерованные пункты (`27.`, `28.`, `1)`, `а)`) → новая строка
3. Ключевые фразы (`Статья`, `Глава`, `Раздел`) → новый блок

## Техническая реализация

```typescript
// В TextContentViewer.tsx

function insertLineBreaks(text: string): string {
  if (!text) return '';
  
  let result = text;
  
  // 1. Пункты типа "27. Текст" или "28. Текст" → перенос перед номером
  result = result.replace(
    /(\.\s*)(\d{1,3})\.\s+([А-ЯЁA-Z])/g,
    '$1\n\n$2. $3'
  );
  
  // 2. Подпункты "1)" или "а)" после текста → перенос
  result = result.replace(
    /([.;:])\s+(\d+\)|[а-яё]\))\s+/gi,
    '$1\n$2 '
  );
  
  // 3. "Статья X.", "Глава X" → новый блок
  result = result.replace(
    /(\.|\s)\s*(Статья\s+\d+|Глава\s+[IVXLCDM\d]+|Раздел\s+[IVXLCDM\d]+)/gi,
    '$1\n\n$2'
  );
  
  // 4. Скобки с редакцией "(Пункт в редакции..." → перенос после
  result = result.replace(
    /(\([^)]*редакции[^)]*\)\.?)\s*/gi,
    '$1\n\n'
  );
  
  return result;
}

function formatChunkContent(text: string): string {
  if (!text) return '';
  
  // Сначала вставляем логические переносы
  let formatted = insertLineBreaks(text);
  
  // Затем применяем HTML-форматирование
  formatted = formatted
    .replace(/&/g, '&amp;')
    .replace(/</g, '&lt;')
    .replace(/>/g, '&gt;');
  
  // Абзацы и переносы
  formatted = formatted.replace(/\n\s*\n/g, '</p><p class="mb-4">');
  formatted = formatted.replace(/\n/g, '<br>');
  
  // Выделение номеров пунктов
  formatted = formatted.replace(
    /(?:^|<br>|<p[^>]*>)(\d{1,3})\.\s/g,
    (m, num) => m.replace(`${num}. `, `<span class="font-semibold text-primary">${num}.</span> `)
  );
  
  // Выделение подпунктов
  formatted = formatted.replace(
    /(\d+\)|[а-яё]\))\s/gi,
    '<span class="text-muted-foreground">$1</span> '
  );
  
  return `<p class="mb-4">${formatted}</p>`;
}
```

## Пример результата

**До (сплошной текст):**
```
...отзванной. 28. Срок представления заявителем ответа на запрос дополнительных материалов, предусмотренный пунктом 22 настоящих Правил, может быть продлен по ходатайству о продлении в соответствии с положениями пункта 54 настоящих Правил. 29. Экспертиза заявленного обозначения проводится...
```

**После:**
```
...отзванной.

28. Срок представления заявителем ответа на запрос 
дополнительных материалов, предусмотренный пунктом 22 
настоящих Правил, может быть продлен по ходатайству о 
продлении в соответствии с положениями пункта 54 
настоящих Правил.

29. Экспертиза заявленного обозначения проводится...
```

## Порядок реализации

1. Добавить функцию `insertLineBreaks()` перед `formatChunkContent()`
2. Вызывать `insertLineBreaks()` первой в цепочке форматирования
3. Улучшить CSS для отступов между абзацами (`mb-4` вместо `mb-3`)
4. Добавить стили для выделения номеров пунктов
