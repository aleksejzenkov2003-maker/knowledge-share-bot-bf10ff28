
Цель: вернуть нормальную прокрутку на странице проекта: чат должен листаться вверх/вниз, длинный текст и JSON не должны обрезаться или “исчезать”, а широкий контент должен либо переноситься, либо скроллиться локально.

Что реально сломано сейчас
- Вертикальная прокрутка чата фактически ломается из-за `scrollIntoView(...)` в `src/components/workflow/WorkflowStepChat.tsx`: при обновлениях он снова тянет список вниз, поэтому пользователь не может нормально подняться вверх.
- Текст “пропадает” не потому, что данных нет, а потому что JSON/структурированный вывод рендерится markdown-ом как `<pre><code>...</code></pre>`, а в `WorkflowStepChat.tsx` для `pre/code` нет нормальной обработки.
- Из-за этого длинный блок выходит по ширине, а родитель с `overflow-hidden` его просто режет.
- В `WorkflowResultEditor.tsx` есть похожий риск: внутренний `pre` уже ограничен, но сам `Card` с `overflow-hidden` может обрезать локальный скролл.

Do I know what the issue is?
Да.

Где причина
- `src/components/workflow/WorkflowStepChat.tsx`
- `src/components/workflow/WorkflowResultEditor.tsx`
- `src/components/workflow/WorkflowStepView.tsx`

Что нужно сделать
1. Исправить автоскролл чата
- Убрать безусловный `scrollIntoView` на каждом обновлении.
- Автопрокрутку оставлять только если пользователь уже возле низа, либо только во время активной генерации.
- Скроллить контейнер сообщений, а не “якорь” внизу.

2. Починить рендер длинного JSON и markdown в чате
- Добавить кастомные markdown-компоненты для `pre` и `code` в `WorkflowStepChat.tsx`.
- Для `pre`: `max-w-full`, локальный `overflow-x-auto`, без расширения родителя.
- Для обычного текста: принудительный перенос (`break-words` / `overflow-wrap:anywhere`).
- Убрать/ослабить `overflow-hidden` на карточке сообщения, если он режет содержимое.

3. Убрать обрезание результата
- В `WorkflowResultEditor.tsx` убрать clipping там, где `Card` мешает прокрутке `pre/table`.
- Оставить горизонтальный скролл только внутри таблиц и code/pre, а не на всём экране.
- Не позволять markdown-контенту растягивать ширину вкладки.

4. Подчистить контейнеры вкладок
- В `WorkflowStepView.tsx` сохранить один понятный scroll-owner на каждую вкладку.
- `chat` — скролл внутри списка сообщений.
- `result`, `structured`, `screenshots` — локальный `overflow-auto` только у контентной области, без лишних промежуточных clip-обёрток.

Как проверю после правки
- В чате можно вручную листать вверх и вниз, и экран не прыгает обратно вниз.
- Длинный JSON не исчезает за правой границей.
- Широкие блоки кода и таблицы либо переносятся, либо скроллятся внутри себя.
- Не появляется двойной общий скролл всей админки.

Технически проблема сейчас выглядит так:
```text
auto-scroll effect in chat
+ JSON/structured text -> markdown <pre>
+ parent overflow-hidden
= нельзя нормально листать вверх, а длинный текст визуально обрезается
```

Следующий фикс должен быть точечным по этим 3 файлам, а не ещё одной общей перестройкой `overflow-hidden/min-w-0`.
