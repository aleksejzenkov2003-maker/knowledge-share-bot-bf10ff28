

## План: Исправить ложные срабатывания PERSON в PII-маскировании

### Проблема

Текущие регулярные выражения для поиска ФИО слишком широкие и маскируют обычные слова как [PERSON_N]. Причины:

1. Флаг `gi` (case-insensitive) -- имена должны начинаться с заглавной буквы, а сейчас ловится всё подряд
2. Суффиксы отчеств слишком короткие: `ич`, `на` -- совпадают с массой обычных слов
3. Падежные паттерны (родительный, дательный, творительный) с суффиксами `а`, `ой`, `у` ловят любые три слова подряд, если третье случайно заканчивается похоже на отчество

### Решение

Упростить до **3 надёжных паттернов** вместо 7 ненадёжных:

1. **Полное ФИО** (именительный) -- строго: Фамилия Имя Отчество, где отчество заканчивается на `ович/евич/овна/евна/ична` (минимум 5 символов суффикса). Без флага `i`.

2. **Фамилия И.О.** -- `Иванов И.И.` или `Иванов И. И.`

3. **И.О. Фамилия** -- `И.И. Иванов` или `И. И. Иванов`

Падежные варианты (родительный, дательный, творительный) **убираем** -- они дают слишком много ложных срабатываний. Основные имена в документах идут в именительном падеже или в формате с инициалами.

### Новые паттерны

```text
// 1. Полное ФИО: Иванов Иван Иванович / Иванова Мария Ивановна
//    Строго: 3 слова с заглавной, третье заканчивается на -ович/-евич/-овна/-евна/-ична
/[А-ЯЁ][а-яё]{1,20}[\s\u00A0]+[А-ЯЁ][а-яё]{1,15}[\s\u00A0]+[А-ЯЁ][а-яё]{3,}(?:ович|евич|овна|евна|ична)/g

// 2. Фамилия И.О.: Иванов И.И. или Иванов И. И.
/[А-ЯЁ][а-яё]{2,20}[\s\u00A0]+[А-ЯЁ]\.[\s\u00A0]?[А-ЯЁ]\./g

// 3. И.О. Фамилия: И.И. Иванов или И. И. Иванов
/[А-ЯЁ]\.[\s\u00A0]?[А-ЯЁ]\.[\s\u00A0]+[А-ЯЁ][а-яё]{2,20}/g
```

### Что убираем

- Все падежные варианты ФИО (родительный, дательный, творительный) -- слишком много ложных срабатываний
- Флаг `i` из всех паттернов -- имена всегда с заглавной буквы
- Короткие суффиксы `ич`, `на`, `вич` -- оставляем только полные: `ович`, `евич`, `овна`, `евна`, `ична`
- Паттерн "Фамилия + Имя + Отчество через запятую"

### Файлы для изменения

| Файл | Изменение |
|---|---|
| `supabase/functions/_shared/pii-patterns.ts` | Заменить 7 паттернов person на 3 строгих, убрать флаг `i` |

После изменения -- переразвернуть edge functions `pii-mask` и `process-document`.

### Ожидаемый результат

- "Бабарыкова Константина Борисовича" -- будет замаскировано (ФИО с отчеством на -овича... нет, мы убрали падежные. Но "Борисовича" заканчивается на "ович" + "а" -- не совпадёт со строгим паттерном)

Стоп, нужно учесть что в документах ФИО часто в родительном падеже. Тогда оставим один расширенный паттерн для полного ФИО, где отчество может быть в любом падеже:

```text
// Полное ФИО (любой падеж отчества):
// ович/овича/овичу/овичем/овиче
// евич/евича/евичу/евичем/евиче  
// овна/овны/овне/овной/овну
// евна/евны/евне/евной/евну
// ична/ичны/ичне/ичной/ичну

/[А-ЯЁ][а-яё]{1,20}[\s\u00A0]+[А-ЯЁ][а-яё]{1,15}[\s\u00A0]+[А-ЯЁ][а-яё]*(?:ович[ауем]?|евич[ауем]?|овн[аыеой]?|евн[аыеой]?|ичн[аыеой]?)\b/g
```

Этот один паттерн покрывает все падежи отчества, но при этом требует строгий суффикс (минимум 4 буквы), что исключает ложные срабатывания.

### Итоговые паттерны (3 штуки)

```text
1. Полное ФИО (все падежи): 
   [А-ЯЁ][а-яё]{1,20} + [А-ЯЁ][а-яё]{1,15} + [А-ЯЁ][а-яё]*(ович/евич/овна/евна/ична + падежные окончания)

2. Фамилия И.О.

3. И.О. Фамилия
```
