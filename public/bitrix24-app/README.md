# Интеграция AI Ассистента с Bitrix24

## Быстрый старт

### Шаг 1: Настройка в вашей системе

1. Откройте **API-ключи** в админ-панели Knowledge Share Bot
2. Создайте новый ключ:
   - **Название**: `Bitrix24 Production`
   - **Отдел**: выберите нужный отдел
   - **Домен Bitrix24 портала**: укажите ваш домен (например: `company.bitrix24.ru`)
3. Сохраните ключ

> ⚠️ **Важно:** API-ключ НЕ передаётся в браузер. Авторизация происходит по домену портала через JWT.

---

### Шаг 2: Создание приложения в Bitrix24

#### 2.1 Перейдите в раздел приложений

1. Откройте ваш портал Bitrix24
2. Нажмите на **Меню** (☰) → **Разработчикам** → **Другое** → **Локальное приложение**

Или прямой URL: `https://ВАШ_ПОРТАЛ.bitrix24.ru/devops/section/standard/`

#### 2.2 Создайте новое приложение

Нажмите **"Добавить приложение"** → **"Серверное приложение"**

#### 2.3 Заполните форму

| Поле | Значение |
|------|----------|
| **Название** | AI Ассистент |
| **Описание** | Интеллектуальный помощник для сотрудников |
| **URL приложения** | `https://knowledge-share-bot.lovable.app/bitrix24-app/index.html` |
| **URL установки** | (оставьте пустым) |
| **Путь для первоначальной установки** | (оставьте пустым) |

#### 2.4 Настройте права доступа

Минимально необходимые права:
- ✅ `user` — Доступ к данным пользователя

#### 2.5 Настройте встраивание (placement)

Нажмите **"Добавить встраивание"** и выберите:

| Параметр | Значение |
|----------|----------|
| **Место встраивания** | `Левое меню` (LEFT_MENU) |
| **Название пункта меню** | AI Ассистент |
| **URL обработчика** | `https://knowledge-share-bot.lovable.app/bitrix24-app/index.html` |

#### 2.6 Сохраните приложение

Нажмите **"Сохранить"**. Приложение будет установлено на ваш портал.

---

### Шаг 3: Проверка работы

1. Обновите страницу Bitrix24
2. В левом меню появится пункт **"AI Ассистент"**
3. Нажмите на него — откроется чат с AI

## Архитектура безопасности

```
┌─────────────────────────────────────────────────────────────┐
│                    Bitrix24 Portal                          │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  iframe: /bitrix24-app/index.html                   │    │
│  │  ├─ BX24.init() → получает placement.DOMAIN         │    │
│  │  ├─ BX24.callMethod('user.current') → user.ID       │    │
│  │  └─ KnowledgeChat.init({ portal, bitrixUserId })    │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│              Knowledge Share Bot (наш сервер)               │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  /bitrix-chat (iframe внутри iframe)                │    │
│  │  ├─ POST /auth { portal, bitrixUserId }             │    │
│  │  │   └─ Ищем API-ключ по portal_domain              │    │
│  │  │   └─ Генерируем JWT (1 час)                      │    │
│  │  │   └─ Сохраняем сессию в bitrix_sessions          │    │
│  │  ├─ Все запросы с Authorization: Bearer <JWT>       │    │
│  │  └─ API-ключ НИКОГДА не покидает сервер             │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

**Безопасность:**
- API-ключ хранится только на сервере
- JWT-токен действует 1 час
- Авторизация привязана к домену портала
- Сессии можно отозвать из админ-панели

## Устранение неполадок

### "Portal not registered"
- Убедитесь, что в настройках API-ключа указан правильный `portal_domain`
- Домен должен совпадать точно: `company.bitrix24.ru`

### "Bitrix24 SDK не найден"
- Приложение открыто вне Bitrix24
- Проверьте что URL приложения доступен по HTTPS

### Пустой экран
- Проверьте консоль браузера (F12)
- Убедитесь что CORS настроен правильно
