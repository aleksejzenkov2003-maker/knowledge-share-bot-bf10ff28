# Интеграция AI Ассистента с Bitrix24

## Быстрый старт

### Шаг 1: Настройка в админ-панели Knowledge Share Bot

1. Откройте **API-ключи** в админ-панели
2. Нажмите **"Создать ключ"**
3. Заполните форму:
   - **Название**: `AI Юрист` (или название вашего отдела)
   - **Отдел**: выберите нужный отдел
   - **Домен Bitrix24 портала**: укажите ваш домен (например: `company.bitrix24.ru`)
4. Сохраните ключ

> ⚠️ **Важно:** Для каждого отдела создайте отдельный API-ключ с одинаковым доменом портала.

---

### Шаг 2: Создание приложения в Bitrix24

#### 2.1 Перейдите в раздел разработчиков

1. Откройте ваш портал Bitrix24
2. Нажмите **Меню** (☰) → **Разработчикам** → **Другое**
3. Выберите **"Локальное приложение"**

Или прямой URL: `https://ВАШ_ПОРТАЛ.bitrix24.ru/devops/section/standard/`

#### 2.2 Заполните форму приложения

| Поле | Значение |
|------|----------|
| **Путь вашего обработчика** | `https://knowledge-share-bot.lovable.app/bitrix24-app/department.html` |
| **Путь для первоначальной установки** | *(оставить пустым)* |
| **Использует только API** | ❌ Не отмечать |
| **Поддерживает BitrixMobile** | ❌ Не отмечать |
| **Название пункта меню** | `AI Юрист` *(или другое название)* |

#### 2.3 Настройте права доступа

В разделе **"Настройка прав"** выберите:
- ✅ **CRM (crm)** — для доступа к данным CRM
- ✅ **Пользователи (user)** — для идентификации пользователя

#### 2.4 Сохраните приложение

Нажмите **"Сохранить"**, затем **"Перейти к приложению"** для проверки.

## Варианты интеграции

### Чат отдела (рекомендуется)

Общий чат для всех сотрудников отдела с автоматическим определением пользователя.

```
URL обработчика: https://knowledge-share-bot.lovable.app/bitrix24-app/department.html
```

### Личный AI-ассистент

Персональный чат с сохранением истории для каждого пользователя.

```
URL обработчика: https://knowledge-share-bot.lovable.app/bitrix24-app/personal.html
```

## Настройка нескольких отделов

Если вам нужно несколько AI-ассистентов для разных отделов на одном портале:

### В админ-панели Knowledge Share Bot:

Создайте API-ключи для каждого отдела:

| Название | Отдел | Домен портала |
|----------|-------|---------------|
| AI Юрист | Юридический | company.bitrix24.ru |
| AI Бухгалтер | Бухгалтерия | company.bitrix24.ru |
| AI HR | HR | company.bitrix24.ru |

### В Bitrix24:

Создайте отдельное локальное приложение для каждого отдела:

| Приложение | Название пункта меню | URL обработчика |
|------------|---------------------|-----------------|
| #1 | AI Юрист | `https://knowledge-share-bot.lovable.app/bitrix24-app/department.html` |
| #2 | AI Бухгалтер | `https://knowledge-share-bot.lovable.app/bitrix24-app/department.html` |
| #3 | AI HR | `https://knowledge-share-bot.lovable.app/bitrix24-app/department.html` |

> 💡 **Как это работает:** URL одинаковый для всех приложений. Система автоматически определяет отдел пользователя по его профилю или назначает первый доступный отдел для новых пользователей.

## Архитектура безопасности

```
┌─────────────────────────────────────────────────────────────┐
│                    Bitrix24 Portal                          │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  iframe: /bitrix24-app/department.html              │    │
│  │  ├─ BX24.init() → получает portal domain            │    │
│  │  ├─ BX24.callMethod('user.current') → user.ID       │    │
│  │  └─ Авторизация через bitrix-chat-api               │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
                               │
                               ▼
┌─────────────────────────────────────────────────────────────┐
│              Knowledge Share Bot (backend)                  │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  Edge Function: bitrix-chat-api                     │    │
│  │  ├─ POST /auth { portal, bitrixUserId }             │    │
│  │  │   └─ Находим API-ключ по portal_domain           │    │
│  │  │   └─ Определяем отдел пользователя               │    │
│  │  │   └─ Генерируем JWT (1 час)                      │    │
│  │  │   └─ Сохраняем сессию                            │    │
│  │  ├─ Все запросы с Authorization: Bearer <JWT>       │    │
│  │  └─ API-ключ НИКОГДА не покидает сервер             │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

**Безопасность:**
- API-ключ хранится только на сервере
- JWT-токен действует 1 час
- Авторизация привязана к домену портала
- Автоматическое определение отдела по профилю пользователя
- Сессии можно отозвать из админ-панели

## Устранение неполадок

### "Portal not registered"

**Причина:** Домен портала не найден в системе.

**Решение:**
1. Проверьте, что в API-ключе указан правильный `portal_domain`
2. Домен должен совпадать точно: `company.bitrix24.ru` (без `https://`)

### "Bitrix24 SDK не найден"

**Причина:** Приложение открыто вне Bitrix24.

**Решение:**
- Убедитесь, что открываете приложение через меню Bitrix24
- Проверьте, что URL приложения доступен по HTTPS

### "Ошибка авторизации"

**Причина:** Истёк JWT-токен или сессия отозвана.

**Решение:**
- Обновите страницу — система автоматически получит новый токен
- Проверьте статус сессии в админ-панели (раздел "Bitrix Сессии")

### Пустой экран

**Решение:**
1. Откройте консоль браузера (F12)
2. Проверьте наличие ошибок в консоли
3. Убедитесь, что CORS настроен правильно

3. Обратитесь к администратору системы
